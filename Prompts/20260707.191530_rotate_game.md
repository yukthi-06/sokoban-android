# User Request
> if the game is wider than height, then rotate the game to fit the free space better

# Implementation Details

To address this request, the game rendering and input handling logic were updated to automatically rotate wide levels by 90 degrees clockwise.

## 1. Game Rendering Update (`GameView.java`)
- The `onDraw` method now checks if `cols > rows`.
- If true, the drawing loop swaps the logical rows and columns, drawing the board rotated by 90 degrees clockwise. This preserves the aspect ratio while allowing a wide board to occupy the taller vertical screen space on mobile devices.

## 2. Input Mapping (`GameActivity.java`)
- Introduced a `getMappedDirection(Direction visualDir)` method.
- When the board is rotated visually, user inputs (swipes or keyboard) are remapped so that they correspond to the correct visual movement on screen.
  - Visual **UP** maps to Logical **LEFT**
  - Visual **DOWN** maps to Logical **RIGHT**
  - Visual **LEFT** maps to Logical **DOWN**
  - Visual **RIGHT** maps to Logical **UP**
