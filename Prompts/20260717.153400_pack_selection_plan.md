# Refactor Main Screen to Show Packs

The main screen currently lists all levels in a single flat list. Now that levels are organized into packs, the main screen needs to act as a pack selector.

## Open Questions
1. **Level Selection Screen**: When a user taps a pack (e.g., "Pack 1 (50)"), do you want it to open a new screen that lists the individual levels for that pack, or should it just immediately start playing the first level of that pack? I'll assume we should create a new `LevelSelectionActivity` for choosing a specific level within the pack, unless you prefer jumping straight into gameplay.
2. **Pack Completion Status**: Should the packs indicate if they are fully completed or show how many levels are completed? (e.g., "Pack 1 (5/50)")

## Proposed Changes

### [New] `PackAdapter.java`
- Create a new adapter to display the list of packs on the main screen.
- Each item will display the pack name and the total number of levels in brackets.

### [Modify] `MainActivity.java`
- Update the main activity to fetch the list of packs from the `LevelRepository`.
- Replace the `LevelAdapter` with the new `PackAdapter`.
- Handle clicks on a pack to open the new level selection screen.

### [Modify] `LevelRepository.java`
- Add a method to retrieve all packs and their corresponding level counts (e.g., `Map<String, Integer> getPacksWithCounts()`).
- Add a method to retrieve all levels for a specific pack (e.g., `List<String> getLevelFilesForPack(String packName)`).

### [New] `LevelSelectionActivity.java` (Optional depending on your feedback)
- Create a new activity similar to the old `MainActivity` that displays the `LevelAdapter` for the specific pack selected.
- Clicking a level here will start the `GameActivity` just like before.

## Verification Plan
- Build and run the application.
- Verify the main screen displays the packs and the correct level counts.
- Tap a pack and ensure it opens the correct list of levels.
- Start a level and ensure gameplay works as expected.
