# Replay Feature & Solved State Plan

The goal is to enhance the gameplay screen when a user revisits a previously solved level. This includes displaying a green "Solved" title, showing the previous best stats, and adding an automatic Replay button.

## User Review Required

- **Replay Button Placement**: I plan to place the "Replay" button right next to the "Undo" and "Restart" buttons in the bottom control bar, making it a 3-button layout. Let me know if you prefer a different layout.
- **Interrupting Replay**: During the automatic replay, should the user be allowed to press "Undo" or touch the screen to instantly stop the replay and take over? I will disable touch controls during the replay by default, but let me know if you want the ability to interrupt it.

## Proposed Changes

### 1. UI Updates (`activity_game.xml`)
- **[MODIFY] activity_game.xml**:
  - Add a new `TextView` (`bestStatsText`) right below the level title to show "Best: X moves, Y pushes, Zs".
  - Add a "Replay" `MaterialButton` into the bottom `LinearLayout` next to "Undo" and "Restart".

### 2. File Reading & Initial State (`GameActivity.java`)
- **[MODIFY] GameActivity.java**:
  - In `loadLevel()`, check if `<GAME_NUMBER>_solution.json` exists in the `solutions` folder.
  - If it exists:
    - Change `levelTitleText` color to green (`#4CAF50`).
    - Append "(Solved)" to the title.
    - Parse the JSON and extract `moves count`, `pushes count`, `timetaken`, and `sequence of moves`.
    - Populate `bestStatsText` and make the "Replay" button visible.
  - If it doesn't exist:
    - Keep standard title color, hide `bestStatsText`, and hide the "Replay" button.

### 3. Replay Engine (`GameActivity.java` & `GameEngine.java`)
- **[MODIFY] GameActivity.java**:
  - Introduce a `Handler` and a `Runnable` loop to act as the replay timer.
  - Create a `boolean isReplaying` flag to block manual touch/keyboard inputs while the replay runs.
  - When "Replay" is clicked:
    - Call `GameEngine.restart()` to reset the level.
    - Start parsing the saved `sequence of moves` string one character at a time.
    - Map characters (`u`, `U`, `d`, `D`, etc.) back to `Direction`.
    - Apply the move via `GameEngine.move()`, update the UI, and schedule the next move in 500 milliseconds.

## Verification Plan
1. Play and beat a new level (e.g., Level 2).
2. Go back to the dashboard and select Level 2 again.
3. Verify that the title says "Level 2 (Solved)" in green, and your stats are shown.
4. Click "Replay" and verify the character automatically solves the level with a 500ms delay between moves.
