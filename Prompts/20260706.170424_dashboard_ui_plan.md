Prompt: the app's main screen should be a dashboard where there are small boxes for level numbers (taken from filename, excluding extn, and trimming spaces and zeroes). On click of each level, user can go to that level and play.

# Dashboard UI Implementation Plan

The goal is to restructure the app so that the main screen (`MainActivity`) serves as a level-selection dashboard displaying levels in a grid of stylized "small boxes". Clicking a box will launch a new `GameActivity` to play the selected level.

## User Review Required

Please review the proposed UI changes below. Since this involves a major shift in how the user navigates the app, I want to ensure the visual design and flow align with your vision.

## Open Questions

- When stripping zeroes, should it remove leading zeroes only (e.g., `007.json` -> `7`, `010.json` -> `10`)? My proposed logic uses a Regex to trim leading zeroes while keeping numbers intact (`fileName.replace(".json", "").replace(".xml", "").trim().replaceFirst("^0+(?!$)", "")`).

## Proposed Changes

### 1. UI Layouts (`sokoban-android/src/main/res/layout/`)

#### [NEW] item_level.xml
- A `MaterialCardView` layout representing a single level box.
- Features `card_bg` as the background, an 8dp corner radius, and an `accent` color text in the center to make it look premium and responsive.
- Will apply a subtle foreground ripple for touch feedback.

#### [MODIFY] activity_main.xml
- Remove the current gameplay layout.
- Replace it with a `RecyclerView` configured to use a `GridLayoutManager` (e.g., 4 columns) to display the level boxes cleanly on the dashboard.

#### [NEW] activity_game.xml
- Essentially the old `activity_main.xml` layout, but the level `Spinner` will be replaced with a static `TextView` showing the current Level name/number.

### 2. Java Code (`sokoban-android/src/main/java/com/sokoban/android/`)

#### [NEW] adapter/LevelAdapter.java
- Standard `RecyclerView.Adapter` that takes the list of filenames.
- Formats the display string by stripping extensions, spaces, and leading zeroes.
- Includes a click listener that fires an event with the selected `levelIndex`.

#### [MODIFY] MainActivity.java
- Remove all game logic (`GameEngine`, `GameView`, swipe controls).
- Keep the runtime permission logic (from the recent SD card update).
- Once permissions are granted, initialize the `RecyclerView` and populate it using the `LevelAdapter`.
- On item click, launch `GameActivity` using an `Intent`, passing the selected `levelIndex`.

#### [NEW] GameActivity.java
- Will contain all the gameplay logic previously housed in `MainActivity` (gestures, undo/restart buttons, victory dialog).
- Retrieves the `levelIndex` from the Intent.
- If the user finishes a level and clicks "Next Level" in the victory dialog, it increments the index and loads the next level internally, or finishes the activity if they completed all levels, returning them to the dashboard.

### 3. App Configuration

#### [MODIFY] AndroidManifest.xml
- Declare `<activity android:name=".GameActivity" />`.

## Verification Plan

- Build the app and confirm the main screen displays a grid of level boxes correctly formatted.
- Tap a level box and verify that the game loads the correct level.
- Complete a level and verify the "Next Level" logic correctly proceeds or returns to the dashboard.
