# User Request
> Between Settings icon and Help icon, add a new icon for 'Index' functionality.
> On clicking this icon, the app indexes all the game levels and creates an index file internally, which helps in identifying the like/dislike status and also completed status.  This will help in displaying the dashboard levels quickly

# Implementation Details

1. **New UI Button (`activity_main.xml`)**:
   - Added an "Index" button (`🗂️`) horizontally placed between the Settings icon and Help icon.

2. **Index Generation (`MainActivity.java`)**:
   - Implemented `buildIndexAsync()`, triggered on `btnIndex` click.
   - Iterates through all levels, checking for `_solution.json` and `like_dislike/*.json` files.
   - Constructs a JSON map containing completion and liked states for each level.
   - Saves to `/sdcard/Vypeensoft/Sokoban/level_index.json`.
   - Runs asynchronously to avoid blocking the main thread, and presents Toast notifications indicating start and completion.

3. **Dashboard Loading Optimization (`MainActivity.java`)**:
   - Updated `initializeDashboard()` to attempt to load `level_index.json`.
   - The index is parsed into memory sets (`completedLevels` and `dislikedLevels`), bypassing the expensive process of enumerating and reading numerous tiny files in the filesystem.
   - If the index isn't found, it seamlessly falls back to the original directory-scanning approach.

4. **Bug Fix: Disliked Levels Filtering**:
   - Previously, the dashboard `RecyclerView` wasn't correctly utilizing the "Show disliked levels" filter.
   - The loading logic was updated to verify if a level was in `dislikedLevels` (via the new index) and skip adding it to the adapter if `showDisliked` was set to false.
