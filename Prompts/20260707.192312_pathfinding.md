# User Request
> from the place where the 'pusher' is standing, he is able to reach many points in the layout.  If the user clicks on any such point, let the pusher quickly reah that point, insead of user having to keep swiping to reach that point.  The actual moves will be recorded in the final json

# Goal Description
Enable "tap-to-move": when the user taps on a reachable point in the layout, the game automatically calculates the path and quickly moves the pusher to that point. The moves must be recorded in the game history just like manual swipes.

## Proposed Changes

1. **Pathfinding Logic (`GameEngine.java`)**
   - Add a `findPath(GameState state, Position target)` method using Breadth-First Search (BFS).
   - The BFS will only traverse `GridCell.EMPTY` and `GridCell.GOAL` cells to ensure the player only walks through free spaces.
   - It will return a `List<Direction>` representing the shortest sequence of moves, or `null` if the destination is unreachable.

2. **Coordinate Translation (`GameView.java`)**
   - Add a `public Position getLogicalPosition(float x, float y)` method.
   - This method will map screen coordinates from a tap back to the logical grid `(row, col)`. It will correctly account for board scaling (`tileSize`), centering (`offsetX`, `offsetY`), and the recently added 90-degree visual rotation.

3. **Tap Detection (`AndroidGameController.java`)**
   - Update `SwipeGestureListener` to override `onSingleTapUp`.
   - Add a new callback `onTap(float x, float y)` to the `OnMoveListener` interface so `GameActivity` receives tap events.

4. **Auto-Move Execution (`GameActivity.java`)**
   - Implement the `onTap` callback to translate the tap to a `Position` and call `findPath`.
   - If a valid path is returned, start a new `autoMoveRunnable` (using a fast interval like 50ms) that sequentially executes each `Direction` using the existing `handleMoveInternal` logic. This ensures the moves are appended to the game history properly.
   - Any manual input (swipe or physical key) during auto-movement will interrupt the sequence.
