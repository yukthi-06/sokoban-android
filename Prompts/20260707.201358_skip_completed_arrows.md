# User Request
> When 'Show completed levels' is false, then while browsing (left and right arrows), it should skip completed levels

# Implementation Details

To ensure the "Show completed levels" preference applies universally across the app, I updated the logic for the Left and Right level navigation arrows.

1. **New Helper Method (`GameActivity.java`)**: 
   - Added a `private boolean isCompleted(int index)` method to easily check if a solution file exists for a given level.

2. **Updated Navigation Logic**:
   - Modified `findNextValidIndex` and `findPrevValidIndex` in `GameActivity.java` to read the `show_completed` preference.
   - While searching for the next/previous level, the loop now checks if `!show_completed` is true and whether the level is completed. 
   - If both are true, it skips that level and continues searching, identical to how it skips disliked levels. 

Now, when you use the header arrows to navigate, it will seamlessly skip over any levels you've already completed if you've chosen to hide them in your settings.
